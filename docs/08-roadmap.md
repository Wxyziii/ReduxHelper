# Development Roadmap

## Phase 1: Planning foundation

Deliverables:

- Project docs
- Feature list
- UI/UX structure
- JSON schemas
- AI prompts
- Data model

Done when:

- An AI coding agent can understand what to build.

## Phase 2: Static UI prototype

Deliverables:

- App shell
- Sidebar
- Top bar
- Dashboard
- Section pages
- File table mock data
- Suggestion cards mock data
- Export page mock data

Done when:

- The app looks and behaves like a real product, but uses mock data.

## Phase 3: Local project storage

Deliverables:

- Create project
- Open project
- Save project JSON
- Section state
- File references
- Project notes

Done when:

- The user can close and reopen a project without losing state.

## Phase 4: File import and scanning

Deliverables:

- Add files
- Add folders
- Drag-and-drop
- Text file reading
- Keyword scanner
- Scan report
- File preview

Done when:

- The app can analyze a folder of exported mod files and group likely matches.

## Phase 5: AI text integration

Deliverables:

- API key setting
- Model setting
- Prompt builder
- AI request
- AI response display
- JSON parsing
- Error handling

Done when:

- The app can send selected snippets to AI and show structured suggestions.

## Phase 6: Patch and diff system

Deliverables:

- Find/replace patch support
- Patch validation
- Diff viewer
- Accept/reject patch
- Edited copy generation

Done when:

- The app can safely generate an edited copy of a text file from accepted AI patches.

## Phase 7: Export system

Deliverables:

- Export folder creation
- Preserve relative paths
- Include backups
- Manifest
- Install notes
- Changelog
- Export summary

Done when:

- The user gets a complete output folder ready for manual import.

## Phase 8: Texture workflow MVP

Deliverables:

- DDS import
- Metadata placeholders
- Preview generation through external converter
- Edited PNG import
- PNG-to-DDS conversion
- Texture export

Done when:

- The app can manage one DDS-to-PNG-to-DDS workflow safely.

## Phase 9: Image AI workflow

Deliverables:

- Local image workflow integration
- Prompt generation
- Edited image result handling
- Preview comparison
- Batch-ready architecture

Done when:

- The app can use an image workflow to produce edited texture previews.

## Phase 10: Polish and safety

Deliverables:

- Better warnings
- Better empty states
- Project history
- Logs
- Error recovery
- Documentation export

Done when:

- The app is stable enough for real modding workflow testing.
