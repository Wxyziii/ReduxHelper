# Project Brief

## App name

Working name: Redux AI Assistant

## Main goal

Build a desktop app that helps the user customize GTA V redux/mod files by splitting the work into clear feature sections such as timecycle, tracers, hit effects, kill effects, optimization, and textures.

The app should not directly edit original game files. It should produce safe exported output folders that the user can review and import manually.

## Core principle

The AI is not the final authority. The AI explains, suggests, and proposes structured edits. The app validates, previews, backs up, and exports.

## Target user

A user who wants to customize visual GTA V files but does not want to manually search through hundreds of files every time.

## Main use cases

- Analyze timecycle files.
- Generate safer edited timecycle values.
- Organize tracer and hit effect files.
- Generate scripting or UI ideas for kill effects.
- Scan optimization-related files.
- Manage texture replacement workflows.
- Convert texture previews through a DDS-to-image-to-DDS pipeline.
- Export ready-to-place files into a new named folder.

## Non-goals for the first version

- Do not build a full OpenIV replacement.
- Do not edit `.rpf` archives directly.
- Do not directly edit binary files without a safe converter/extractor.
- Do not auto-apply hundreds of AI changes.
- Do not promise perfect in-game results.
- Do not bypass game protections or anti-cheat systems.
- Do not include cheating features, gameplay automation, wallhacks, memory readers, or competitive advantage tools.
